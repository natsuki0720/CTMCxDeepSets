# NPE化 成果まとめ（2026-06-12 時点）

**対象:** CTMC 点推定サロゲート（DeepSets 回帰）→ 償却ベイズ推定（NPE）への拡張。
**到達点:** **ガウス構造√Kヘッドで production GO（多面検証済み）。** サンプルサイズ依存の
事後収縮を「学習」ではなく「構造」で保証し、点推定精度を MLE サロゲート据え置き〜改善のまま、
較正された事後分布を獲得。`--no-logk-context` の優位を同条件 A/B で確定。

関連: [設計書](NPE化_差分実装設計書.md)（§10 が本作業の設計）／
[構造化√K 検証サマリ](NPE化_構造化√K_検証サマリ.md)（〜struct_gauss2 の詳細）。本書は
それ以降（K_REF・本番run・A/B・MLE比較）を含む統合版。

## 1. 何を作ったか

- 点推定サロゲート → **償却ベイズ（NPE）** に拡張。自己完結パッケージ `src/ctmc_surrogate`
  ＋ CLI（`scripts/train_entrypoint.py` / `scripts/evaluate_npe.py`）。
- ガウス／flow の**構造化ヘッド**：`z = μ(pooled) + u/√K`。1/√K 収縮をアーキに焼き込み。
- 評価モジュール：複製テスト・K層別 SBC・被覆・収縮傾き・厳密事後との照合・MLE/真値比較。

---

## 2. 核心の科学的成果 — 1/√K を構造で入れる

対数尤度 `ln L = K·⟨ln p⟩` ゆえ対数事後 Hessian は厳密に `K×(経験曲率)`。**p̂ を固定すれば
事後幅 ∝ 1/√K は恒等式**で、学習対象ではない。512次元 pooled に 1次元 log K を混ぜて学ばせる
設計書原案は信号が弱すぎ**複製テストを落とす**（run `big_01`: 複製比 1.018／傾き≈0）。

→ アーキに `z = μ(pooled) + u/√K` を焼き込み。成立に必要だった3点（詳細は検証サマリ §4）：
1. μ を **detach＋補助MSE** で学習（NLL の μ 勾配は精度∝K で病的）。
2. エンコーダを scale/shape NLL から隔離（`pooled.detach()`）。
3. **`mu_head` を MLP 化**（512→128→64→3）← 最重要。Linear は μ-RMSE 0.34 で頭打ち、MLP 0.14。

**K_REF 修正（設計書 §10.7, run `struct_gauss3`）:** 残差標準化を K=1 でなく `k_ref`（=1000）基準に
（`scale=√(k_ref/K)`、定数なので複製/傾き不変）。弱識別 state でも u~3 に収まり、ガウスクランプ/
NSF 域を踏み外さない。弱ストラタム（visits<50）の 90%被覆 0.823→0.935。`--grad-clip 5.0` 併設。

## 4. logk_context A/B 決着（2026-06-12）

`ab_logk`（logk 文脈あり）vs `ab_nologk_01`（`--no-logk-context`、純・構造√K）を**同条件で 1:1 比較**。

| 指標（目標） | logk **有** | logk **無**（本命） | 判定 |
|---|---|---|---|
| 複製比（0.707） | 0.778 過分散 | **0.707 ピタリ** | ◎ 無 |
| 傾き dim0/1/2（−0.5） | −0.37/−0.33/−0.39 甘い | **−0.48/−0.50/−0.50** | ◎ 無 |
| 被覆 .5/.9 | 0.53/0.95 | 0.53/0.92 | ＝ |
| emulation Δmean z | 0.057 | **0.045** | ○ 無 |
| recovery RMSE vs真値 | 0.124 | 0.122 | ＝ |
| SBC-KS dim0/1/2 | 0.143/0.058/**0.076** | **0.086**/0.060/0.166 | △ dim2 で無が悪化 |

**結論:** **`--no-logk-context` の優位を確定。** 設計書 §10.8 の狙い通り、log K をヘッド文脈に
入れると複製が過分散側（0.778）へ膨らみ傾きが −0.33 に引きずられるが、外すと**複製 0.707・
傾き ≈−0.5 と理論値ピタリ**、点推定の emulation もむしろ改善。唯一 **SBC dim2（KS 0.166）が悪化**
＝1次元の較正の皺（Phase 2 / dim2 個別点検の対象）。

## 5. NPE-MAP は MLE サロゲートより悪化していない（点推定の検証）

「NPE 化でピーク（MAP）の当て精度が落ちないか」を、`notebooks/result__likelihood_*.csv`
（`pred`=サロゲート出力 / `likelihood_lifespan`=数値MLE / `true`=真値）で**3者分解**。ガウスヘッドゆえ
posterior mean = mode = MAP。単位は z = log ν、健全部分集合（発散MLE除外後）。

| 量 | K=5000 | K 500-5000 |
|---|---|---|
| 統計床：数値MLE vs 真値 | 0.037 | 0.064 |
| 回収：サロゲート vs 真値 | 0.058 | 0.072 |
| **emulation：サロゲート vs 数値MLE** | **0.049** | **0.061** |
| 参考：NPE-MAP vs 解析的事後平均（emulation） | 0.034〈gachi〉/ 0.057〈ab_logk〉 | 同 |
| **数値MLE が inf/0 に発散した割合** | **9.7%** | **7.5%** |

**結論:**
1. **ピーク精度は劣化していない。** サロゲートが MLE を償却する誤差 0.049–0.061 に対し、NPE が
   事後平均（=MAP）を償却する誤差 0.034–0.057。同等〜規模が出れば NPE が良い。
2. 従来の「サロゲート 0.080」は **vs 真値**で統計床（MLE自体のばらつき）を含み emulation を過大評価。
   純 emulation は ~0.05。回収 0.058 ≒ √(統計床0.037² + emulation0.049²) と整合。
3. **質的優位:** 真値の **7.5–9.7% で数値MLEが発散**（弱識別 state、Dirichlet が単体辺を踏む）し点推定が
   定義されない。NPE はそこでも有限の MAP＋較正された幅を返す。**約1割の領域で MLE が壊れるが NPE は答える。**

## 6. 残課題

- **SBC dim2 の較正ズレ**（`--no-logk-context` で KS 0.166）— logk 除去が3次元目の幅に影響。
  Phase 2（flow / μ warm-up）か dim2 個別点検。
- **弱ストラタム 50%被覆 ~0.39** — ガウスでは非ガウスな広posterior形状を表現不可。flow 待ち。
- **flow ヘッド未達（Phase 2）** — NSF 有界スプライン域が μ 過渡期の O(√K) 残差を保持できず。
  μ warm-up か NSF `bound` 拡大が必要。当面ガウス推奨。
- **Phase B（未着手）** — 訓練K帯を log-uniform [100,5000] に拡張（低K SBC＋クリーンOOD評価）。
- 実データ評価（設計書 §4.3）— 事後平均/中央値 vs v2点推定・MLE、事後SD vs MLE漸近SD。

## 7. 運用メモ（再現・落とし穴）

- **本実行構成:** `--head gaussian --sqrt-k-scaling --k-ref 1000 --no-logk-context`
  （対照 ablation は `--no-sqrt-k-scaling --logk-scale 1.0`、または `--logk-context`）。
- **環境:** conda env `ctmc`（torch 2.5.1 + zuko 1.6.0）、`--device cuda`、GPU共有で batch≤64、
  `PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True`。データは外付け `/mnt/ssd/datas/`。
- **stdout バッファリング:** `conda run` 経由だと epoch ログが全バッファ化して見えない。
  `PYTHONUNBUFFERED=1 ... > log 2>&1` か `tee` でログ化（ライブ監視可・死因も残る）。
- **auto-run-dir の _NN サフィックス（重要）:** `<run-name>/` が既存だと上書きせず `<run-name>_01`
  に保存する。中断で残った空dirがあると成果物がそちらに逸れ、固定パスの eval が
  `model_config.yaml not found` で落ちる（学習は成功している）。**再起動前に空dirを削除**、または
  `--no-auto-run-dir`、または eval は run_config.json の `resolved_run_dir` を参照すること。
  実例: 本セッションの `ab_nologk` は `ab_nologk_01` に保存された。
- **進捗の観測:** ログが無くても `py-spy dump --pid <pid>` で生存・実行位置を確認可
  （GPU律速時は `_run_epoch` の `loss.item()` 同期点に滞留＝正常）。1エポット ≈ 30–50s（50k subset）。
